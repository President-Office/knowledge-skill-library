# Elasticsearch

## 核心定位

Elasticsearch 是面向搜索和日志检索的分布式搜索引擎。它常用于全文搜索、日志分析、监控检索和复杂过滤查询。

## 适合场景

- 商品、文章、客户、工单等全文搜索。
- 日志、审计、监控数据检索。
- 多字段过滤、排序、高亮、分词和相关性排序。

## 学习重点

| 主题 | 为什么重要 |
| --- | --- |
| 倒排索引 | 搜索引擎和关系型数据库索引的核心差异 |
| Mapping | 字段类型、分词器和查询效果强相关 |
| 分词 | 中文搜索尤其依赖分词策略 |
| 同步策略 | 业务数据库和搜索索引之间需要同步 |
| 最终一致性 | 搜索索引通常不是强一致主数据源 |

## 在当前项目中观察

- [[payloadcms--payload]]：搜索插件可以作为观察后台系统如何接入搜索能力的入口。
- [[CRM]]、商城、内容管理类项目通常会引入搜索索引来提升查询体验。

## 不要误解

- Elasticsearch 不是关系型数据库替代品。
- 搜索索引通常不应该作为唯一业务事实来源。
- 搜索效果依赖 mapping、分词、同步和查询设计，不是接入后自动变好。
