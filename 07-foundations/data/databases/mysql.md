# MySQL

## 核心定位

MySQL 是 Web 后台系统中最常见的通用关系型数据库之一。它适合学习 [[oltp]]、表结构设计、事务、索引、SQL 查询、主从复制、分库分表和常规后台数据建模。

## 适合场景

- 中后台系统、SaaS、CRM、ERP 的常规业务数据。
- 用户、角色、菜单、订单、支付、商品、流程等强结构化数据。
- 需要事务和关系约束，但不一定需要复杂数据库扩展能力的应用。

## 学习重点

| 主题 | 为什么重要 |
| --- | --- |
| 表结构设计 | 后台系统的业务边界最终会落到表、字段和关系 |
| 索引 | 大多数性能问题首先要看查询条件和索引 |
| 事务 | 支付、订单、库存、账户类业务必须理解事务边界 |
| 慢查询 | 真实系统调优离不开执行计划和慢 SQL |
| 多租户 | 常见做法是租户字段、库表隔离或数据权限过滤 |

## 不要误解

- MySQL 常见不等于简单，索引、事务隔离、锁和 SQL 设计都会影响系统稳定性。
- ORM 不能替代数据库设计。
- 分库分表不是默认选项，先把数据模型和访问模式弄清楚。
