# SQLite

## 核心定位

SQLite 是嵌入式、单文件、零服务进程的关系型数据库。它适合本地应用、移动端、桌面软件、小型工具、测试和轻量 [[oltp]] 原型验证。

## 适合场景

- 单机应用、本地缓存、本地配置和轻量数据存储。
- 测试环境或 demo 项目。
- 数据规模和并发写入压力不高的工具型应用。

## 学习重点

| 主题 | 为什么重要 |
| --- | --- |
| 单文件数据库 | 部署、备份和迁移非常简单 |
| 嵌入式运行 | 不需要单独数据库服务 |
| SQL 基础 | 仍然可以学习表、索引、查询、事务 |
| 并发边界 | 不适合高并发写入场景 |

## 在当前项目中观察

- [[payloadcms--payload]]：SQLite 常用于模板、开发环境或轻量部署选项。
- 本地实验页：适合先用 SQLite 跑通概念，再切换到 PostgreSQL 或 MySQL。

## 不要误解

- SQLite 不是玩具数据库，但它的并发和运维边界不同于服务端数据库。
- 它适合小而稳的场景，不适合所有后台生产系统。
