# 外键

## 核心定位

外键是表中引用另一张表主键或唯一键的字段，用来表达表之间的关系。

一句话理解：

**外键回答“这一行和另一张表的哪一行有关”。**

例如：

```text
major.school_id -> school.id
volunteer_plan.student_id -> student_profile.id
```

## 外键表达的关系

| 关系 | 例子 |
| --- | --- |
| 一对多 | 一所院校有多个专业 |
| 多对一 | 多个专业属于同一所院校 |
| 多对多 | 学生收藏多个专业，专业也被多个学生收藏 |
| 明细归属 | 一个志愿方案有多个志愿项 |

多对多通常需要中间表表达：

```text
student_favorite_major
  student_id
  major_id
```

## 数据库外键约束和逻辑外键

有些系统会真的创建数据库外键约束；有些系统只在代码里维护关联字段。

| 做法 | 优点 | 代价 |
| --- | --- | --- |
| 数据库外键约束 | 数据库层阻止孤儿数据 | 迁移、批量导入、分库分表更复杂 |
| 逻辑外键 | 开发和扩展更灵活 | 需要代码和数据治理保证一致性 |

很多国内后台脚手架会偏向逻辑外键，但这不等于可以忽略关系建模。

## 常见误区

- 外键不是只有数据库约束才算关系；字段语义和代码依赖也会形成关系。
- 不要用一个字段同时引用多种表，除非明确采用多态关联并能治理复杂度。
- 删除主表数据前，要考虑从表数据如何处理：禁止删除、级联删除、软删除还是保留历史。
