# 对象

## 核心定位

对象是程序运行时存在的具体实体，它有状态，也可以有行为。

一句话理解：

**类描述“有什么样的东西”，对象是运行时真正产生的“这个东西”。**

例如：

```text
Class: Student
Object: 张三这个学生
```

## 对象包含什么

| 内容 | 含义 | 例子 |
| --- | --- | --- |
| 身份 | 这个对象和另一个对象是否是同一个 | studentId、内存引用、数据库主键 |
| 状态 | 当前对象保存的数据 | 分数、省份、排名、志愿偏好 |
| 行为 | 对象能执行的操作 | updateScore、chooseMajor、submitPlan |
| 约束 | 什么操作合法，什么操作非法 | 已确认志愿不能随意修改 |

不是所有语言都会把这些都放在“对象”里，但面向对象的核心思路是把状态和行为组织在一个有边界的单位中。

## 对象如何产生

对象通常来自某个类型描述，例如 [[class]]、[[struct]]、record、prototype 或工厂函数。

典型路径：

```text
类型描述 -> 构造/初始化 -> 运行时对象
```

更完整的理解见：[[from-struct-to-class]]。

## 对象和数据库记录

后台项目里经常会看到两种“对象”：

| 类型 | 说明 |
| --- | --- |
| 内存对象 | Java / Go / TypeScript 运行时里的对象 |
| 持久化对象 | 数据库里的一行记录，通常通过 [[orm]] / Mapper 映射为内存对象 |

不能简单把数据库记录等同于业务对象。数据库记录偏持久化，业务对象还应该表达行为、约束和状态变化。

## 常见误区

- 对象不只是字段集合；如果只有字段没有约束，它更像数据结构。
- 类不是对象；类是描述，对象是运行时实例。
- 数据库表的一行不自动等于领域对象，还要看是否承载业务语义。
