# ERP

## 核心定位

ERP 是 Enterprise Resource Planning，企业资源计划系统。

一句话理解：

**ERP 负责把企业里的采购、销售、库存、财务、生产、人力等资源和交易流程放到一套一致的数据体系里管理。**

ERP 的重点不是“后台页面很多”，而是跨部门业务事实的一致性。

## ERP 管什么

| 领域 | 典型对象 |
| --- | --- |
| 财务 | 科目、凭证、应收、应付、总账 |
| 采购 | 供应商、采购订单、入库、发票 |
| 销售 | 客户、销售订单、出库、收款 |
| 库存 | 仓库、物料、批次、库存流水 |
| 生产 | 物料清单、生产订单、成本 |
| 组织 | 公司、部门、员工、权限 |

不同 ERP 产品覆盖范围不同，但财务、库存、采购、销售通常是核心。

## ERP 的核心难点

ERP 的难点在于业务链路互相影响：

```text
采购 -> 入库 -> 库存 -> 应付
销售 -> 出库 -> 库存 -> 应收
生产 -> 领料 -> 完工 -> 成本
```

一个动作往往会影响多个模块和多张表。

所以 ERP 非常依赖：

- [[database-transaction]]
- 业务流水
- 审计追踪
- 状态机
- 权限和组织结构
- 主数据治理

## ERP 和普通后台的区别

普通后台可能只是 CRUD，ERP 强调跨模块业务一致性。

| 普通后台 | ERP |
| --- | --- |
| 单模块数据管理 | 跨部门流程协同 |
| 页面和表驱动 | 交易、流水和财务一致性驱动 |
| 修改数据相对简单 | 状态、审计、反冲、冲销更重要 |
| 可以较轻 | 业务规则重，数据后果重 |

## 常见误区

- ERP 不是“把所有后台模块放一起”。
- ERP 的核心不是页面数量，而是跨模块业务事实和一致性。
- ERP 不一定适合快速随意改字段，因为生产数据迁移和财务后果很重。
