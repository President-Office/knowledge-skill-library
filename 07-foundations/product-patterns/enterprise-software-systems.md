# 企业软件系统

## 核心定位

企业软件系统是围绕企业经营、生产、销售、内容、客户、财务和协作流程构建的软件系统。

一句话理解：

**不同企业系统的区别，不只是页面不同，而是它们管理的业务对象、流程边界和数据事实不同。**

## 常见类型

| 类型 | 全称 | 核心管理对象 |
| --- | --- | --- |
| [[cms]] | Content Management System | 内容、栏目、页面、媒体、发布流程 |
| [[erp]] | Enterprise Resource Planning | 财务、采购、销售、库存、生产、组织资源 |
| [[mes]] | Manufacturing Execution System | 车间、工单、工序、设备、产线、质量、生产执行 |
| [[crm]] | Customer Relationship Management | 客户、线索、商机、跟进、合同、销售过程 |

这些系统经常互相集成，但它们不是同一个东西。

## 判断一个系统属于哪类

可以先问三个问题：

- 它最核心的业务事实是什么？
- 它主要服务哪个组织角色？
- 它管理的是计划、执行、交易、内容，还是关系？

例如：

| 如果系统重点是 | 更接近 |
| --- | --- |
| 内容编辑、审核、发布、多端 API | [[cms]] |
| 采购、销售、库存、财务、组织资源协同 | [[erp]] |
| 工单下发、工序执行、设备采集、质量追踪 | [[mes]] |
| 客户跟进、销售机会、商机转化、合同关系 | [[crm]] |

## 和后台框架的关系

RuoYi/Yudao、JeecgBoot、gin-vue-admin 这类后台框架本身不是 ERP、CRM 或 MES。

它们更像通用后台底座，提供：

- 用户、角色、权限、菜单。
- 代码生成、表单、字典、配置。
- 工作流、通知、日志、文件、支付等通用能力。

真正的 ERP、MES、CRM 还需要具体业务模型和流程沉淀。

## 和数据库的关系

企业软件系统的核心通常落在 [[database]] 和 [[schema]] 上：

- CMS 关注内容模型和发布状态。
- ERP 关注交易流水和财务一致性。
- MES 关注生产过程和追溯链路。
- CRM 关注客户关系、销售阶段和跟进记录。

所以学习企业软件，不能只看页面，要看表、状态机、流程和数据约束。
