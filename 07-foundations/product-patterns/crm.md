# CRM

## 核心定位

CRM 是 Customer Relationship Management，客户关系管理系统。

一句话理解：

**CRM 负责管理客户从线索、跟进、商机、成交到续费和服务的关系过程。**

CRM 的核心不是“客户表”，而是客户关系和销售过程。

## CRM 管什么

| 对象 | 说明 |
| --- | --- |
| 线索 | 潜在客户来源和初步信息 |
| 客户 | 已确认的个人或企业客户 |
| 联系人 | 客户组织内具体沟通对象 |
| 商机 | 某次可能成交的销售机会 |
| 跟进记录 | 电话、微信、拜访、会议、备注 |
| 合同 / 订单 | 成交后的商业结果 |
| 销售阶段 | 初访、需求确认、报价、谈判、成交、流失 |
| 客户服务 | 售后、工单、续费、满意度 |

## CRM 的核心难点

CRM 的难点在于过程数据和责任归属：

- 线索从哪里来？
- 谁负责跟进？
- 商机处于哪个阶段？
- 为什么成交或流失？
- 客户是否重复、撞单或归属不清？
- 销售过程是否可追踪？

这意味着 CRM 经常要处理权限、数据归属、销售漏斗、跟进提醒、客户去重和报表。

## CRM 和 ERP 的关系

可以粗略理解为：

```text
CRM：成交前和客户关系过程
ERP：成交后和企业资源/交易履约
```

典型链路：

```text
CRM 线索 -> 商机 -> 合同
ERP 销售订单 -> 出库 -> 应收 -> 财务
```

实际系统里二者会集成，但关注点不同。

## CRM 和普通客户管理的区别

| 普通客户表 | CRM |
| --- | --- |
| 保存客户资料 | 管理客户生命周期 |
| 关注客户字段 | 关注线索、商机、跟进和转化 |
| 权限相对简单 | 强调客户归属、销售团队、数据可见性 |
| 报表较简单 | 销售漏斗、转化率、业绩预测更重要 |

## 常见误区

- CRM 不只是客户档案，它重点管理客户关系过程。
- CRM 不等于 ERP 销售模块，CRM 更偏成交前和客户生命周期。
- 没有数据归属和跟进记录的客户表，很难支撑真正销售管理。
