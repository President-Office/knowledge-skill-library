# MES

## 核心定位

MES 是 Manufacturing Execution System，制造执行系统。

一句话理解：

**MES 负责把生产计划落到车间执行过程里，并记录工单、工序、设备、人员、质量和追溯数据。**

ERP 更偏企业资源和计划协同，MES 更贴近生产现场执行。

## MES 管什么

| 对象 | 说明 |
| --- | --- |
| 工单 | 要生产什么、数量多少、什么时候生产 |
| 工序 | 生产过程拆成哪些步骤 |
| 产线 / 工位 | 在哪里生产 |
| 设备 | 哪些设备参与生产，状态如何 |
| 人员 | 谁执行、谁检验、谁确认 |
| 物料 | 投入了哪些批次和数量 |
| 质量 | 检验项、不良品、返工、报废 |
| 追溯 | 产品从原料到成品的过程记录 |

## MES 的核心难点

MES 的难点在于“现场事实”的准确记录：

- 工单是否按计划执行。
- 工序是否按顺序完成。
- 设备状态是否可信。
- 物料批次能否追溯。
- 质量异常能否定位到人、机、料、法、环。

这些数据往往来自人工录入、扫码、设备采集、传感器、PLC、看板等多种来源。

## MES 和 ERP 的关系

可以粗略理解为：

```text
ERP：计划和资源
MES：执行和现场
```

典型链路：

```text
ERP 下达生产订单 -> MES 拆解工单和工序 -> 车间执行 -> MES 回传产量、质量、物料消耗 -> ERP 更新库存和成本
```

它们关系紧密，但边界不同。

## MES 和普通后台的区别

| 普通后台 | MES |
| --- | --- |
| 人工 CRUD 为主 | 现场执行和设备数据更重要 |
| 数据更新频率较低 | 生产现场可能高频采集 |
| 业务流程相对离散 | 工序、工单、质量、追溯链路强 |
| 对实时性要求可低 | 部分场景要求准实时看板和异常响应 |

## 常见误区

- MES 不是 ERP 的一个简单页面模块，它贴近车间执行。
- MES 不是只有设备联网，工单、工序、质量和追溯同样关键。
- 没有清晰物料、工序、批次模型时，MES 很容易变成录入系统。
