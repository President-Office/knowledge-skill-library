# Projects

每个开源项目一张主卡，用来回答“这个项目是什么、为什么值得学、源码从哪里切入、最后学到了什么”。

## 建卡标准

使用 `_template-open-source-project.md` 复制新文件，命名为：

```text
<owner>--<repo>.md
```

例如：

```text
directus--directus.md
supabase--supabase.md
ruanyf--weekly.md
```

## 项目卡应该包含

- GitHub 地址、许可证、技术栈、活跃度。
- 项目定位和适合学习的原因。
- 本地运行和调试入口。
- 目录结构和核心模块。
- 关键调用链、数据模型、扩展点。
- 和同类项目的差异。
- 可复用经验和后续问题。
