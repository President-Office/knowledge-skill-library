# Patterns

这里保存从多个开源项目中提炼出的可复用工程模式。只有跨项目有价值的经验才放进来。

## 适合放入的内容

- 架构模式：插件系统、权限模型、多租户、任务调度、事件驱动。
- 代码组织：模块边界、目录结构、领域服务、适配器设计。
- 工程实践：测试策略、CI/CD、配置管理、版本发布。
- 产品实现：后台管理、工作流、低代码、CMS、支付、通知。

## 不适合放入的内容

- 某个项目的临时运行报错。
- 只对单一项目有效的目录说明。
- 没有证据支撑的泛泛结论。
