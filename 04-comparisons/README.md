# Comparisons

这里保存同类开源项目、框架、工具和技术路线的横向比较，用来支持选型判断。

## 推荐写法

- 先给结论，再给证据。
- 明确比较维度，不使用会随时间变化的热度和短期活跃度指标作为主要依据。
- 区分项目定位、技术栈、部署复杂度、扩展性、许可证、生态和学习价值。
- 优先沉淀长期有效的架构差异、许可证差异、学习路线和选型判断。
