# Reading Notes

这里保存源码阅读笔记。笔记应该围绕具体模块、调用链、功能点、Issue、PR 或技术问题，不要变成泛泛摘要。

## 推荐目录

```text
02-reading-notes/
└── <owner>--<repo>/
    ├── startup-flow.md
    ├── auth-module.md
    ├── data-model.md
    └── plugin-system.md
```

## 记录要求

- 写清楚阅读目标：为什么读这一段。
- 写清楚证据：源码路径、函数名、配置文件、运行结果。
- 写清楚结论：这段代码如何工作，有什么优缺点。
- 最后沉淀复用点：能否进入 `03-patterns/`。
